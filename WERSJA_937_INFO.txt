WERSJA 937

Dodano system ręcznego rozliczania kuponów z zatwierdzeniem admina.

Zrobione:
1. Ręcznie dodany typ użytkownik może zgłosić jako:
   - won / wygrany
   - lost / przegrany
   - void / zwrot
2. Zgłoszenie NIE dopisuje od razu statystyk.
3. Typ dostaje status:
   - status: pending_admin
   - admin_approval_status: pending
   - manual_settlement_result: wynik zgłoszony przez użytkownika
4. Admin ma nową zakładkę tylko dla admina:
   - Kupony admin
5. W zakładce admin widzi:
   - Czekają
   - Zatwierdzone
   - Odrzucone
6. Admin może kliknąć:
   - Potwierdź wygraną
   - Potwierdź przegraną
   - Zwrot
   - Odrzuć
7. Dopiero po zatwierdzeniu admina status zmienia się na won/lost/void i może liczyć się do statystyk.
8. Dodałem SQL migrację:
   - supabase/version_937_manual_coupon_approval.sql
