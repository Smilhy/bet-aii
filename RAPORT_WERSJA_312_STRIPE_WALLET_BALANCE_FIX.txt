Wersja 312 - Stripe wallet balance fix

Naprawione:
- po powrocie ze Stripe Checkout aplikacja wywołuje /.netlify/functions/sync-wallet-session,
- funkcja serwerowa sprawdza sesję Stripe po STRIPE_SECRET_KEY,
- jeśli płatność jest paid/complete, zapisuje transakcję wallet_transactions jako completed,
- saldo w UI odświeża się natychmiast po powrocie ze Stripe,
- webhook stripe-webhook używa tej samej logiki, więc działa też gdy webhook jest poprawnie podłączony,
- SQL version_312_stripe_wallet_balance_fix.sql tworzy/naprawia wallet_transactions i get_wallet_balance.

Po wdrożeniu:
1. Uruchom supabase/version_312_stripe_wallet_balance_fix.sql w Supabase (Run without RLS).
2. Upewnij się, że Netlify ENV ma STRIPE_SECRET_KEY, SUPABASE_URL/VITE_SUPABASE_URL i SUPABASE_SERVICE_ROLE_KEY.
3. Dla webhooka w Stripe dodaj endpoint: https://TWOJA-DOMENA/.netlify/functions/stripe-webhook i ustaw STRIPE_WEBHOOK_SECRET.
4. Test: doładuj 100 zł kartą testową Stripe 4242 4242 4242 4242, saldo powinno wskoczyć po powrocie.
