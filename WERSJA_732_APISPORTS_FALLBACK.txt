WERSJA 732 — API-SPORTS FALLBACK

Co zmieniono:
1. The Odds API nadal jest głównym źródłem kursów.
2. Dodano API-Sports jako fallback do realnych meczów/eventów.
3. Backend czyta zmienne:
   - APISPORTS_KEY
   - API_FOOTBALL_KEY
   - API_SPORTS_KEY
   - ODDS_API_KEY / THE_ODDS_API_KEY
4. Jeśli The Odds API nic nie zwróci, backend automatycznie próbuje API-Sports.
5. Obsługiwane fallbacki API-Sports:
   - Football: v3.football.api-sports.io/fixtures
   - Basketball: v1.basketball.api-sports.io/games
   - NBA: v2.nba.api-sports.io/games
   - Baseball: v1.baseball.api-sports.io/games
   - Hockey: v1.hockey.api-sports.io/games
   - Volleyball: v1.volleyball.api-sports.io/games
   - Handball: v1.handball.api-sports.io/games
   - NFL/American Football: v1.american-football.api-sports.io/games
   - Rugby: v1.rugby.api-sports.io/games
   - AFL: v1.afl.api-sports.io/games
   - MMA: v1.mma.api-sports.io/fights
6. Boks zostaje głównie przez The Odds API, bo API-Sports dashboard nie pokazuje osobnego Boxing API.
7. Nie dodano fake/demo meczów — fallback pokazuje tylko dane zwrócone z API.
