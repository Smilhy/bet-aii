WERSJA 770 — automatyczny zapis meczów po resecie limitu
- Bazuje na wersji 765 użytkownika
- Dodana Netlify Scheduled Function: refresh-fixture-cache.js
- Domyślny harmonogram: codziennie 00:05 UTC, czyli 5 minut po dziennym resecie limitu API-Football
- Domyślnie pobiera i zapisuje mecze na 7 dni do przodu; można zmienić ENV FIXTURE_DAILY_PREFETCH_DAYS
- "Dodaj typ" najpierw korzysta z zapisanych meczów; API pyta dopiero gdy cache jest pusty
- "Typy AI" najpierw korzystają z zapisanych meczów; silnik AI też czyta football cache przed pobieraniem z API
- Gdy API ma limit lub chwilowo nie odpowiada, strona pokazuje zapisane mecze z cache
- Do wdrożenia:
  1) Uruchom SQL BETAI_SQL_770_AUTO_DAILY_FIXTURE_CACHE_DODAJ_TYP_AI.sql w Supabase SQL Editor
  2) Wgraj ZIP na Netlify
  3) Po deployu w Netlify > Functions powinieneś zobaczyć refresh-fixture-cache z oznaczeniem Scheduled
  4) Możesz kliknąć Run now w Netlify Functions, żeby ręcznie zapełnić cache po odnowieniu limitu
